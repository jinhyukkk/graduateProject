## 초록 

기업 현장에서 데이터 활용 수요가 커지면서, 자연어 질문을 데이터베이스 질의어로 자동 변환하는 Text-to-SQL 기술이 주목받고 있다. 특히 변환된 질의문의 오류를 스스로 점검·수정하는 자기교정 기법이 중요해지고 있다. 그러나 기존 연구는 질의문이 실제로 실행되는지, 실행 결과가 서로 일치하는지 등 코드 실행 측면에 집중되어 있다. 이 때문에 질의문이 오류 없이 실행되더라도 사용자의 원래 의도와 다른 결과를 돌려주는 "의도 불일치" 문제를 잡아내기 어렵다. 본 연구는 두 가지 질문에 답하고자 한다. 첫째, 생성된 질의문을 다시 자연어로 바꿔 원 질문과 의미가 같은지 확인하는 자연어 추론(Natural Language Inference, NLI) 기반 검증이, 기존의 실행 기반 검증과 별개로 성능 향상에 기여하는가? 둘째, 진단된 오류 유형별로 서로 다른 교정 지시문을 사용하는 방식이, 하나의 공용 교정 지시문을 쓰는 방식보다 더 정확한 교정을 이끌어내는가? 이를 위해 본 연구는 자기교정 Text-to-SQL 프레임워크를 제안하며, 그 핵심으로 의도 일치 검증기를 둔다. 이 검증기는 생성된 질의문을 자연어 문장으로 역번역한 뒤, 자연어 추론 모델을 이용해 원 질문과 의미가 일치하는지 판정한다. 이어 결과를 사전에 정의된 오류 유형으로 분류하고, 유형에 맞는 전용 교정 지시문으로 연결한다.
평가는 두 단계로 진행한다. 먼저 대규모 벤치마크 BIRD(대규모 Text-to-SQL 벤치마크) 검증셋에서 선행 모델인 DAIL-SQL, MAC-SQL을 동일한 대형 언어모델(GPT-4o)로 재실행해 공정한 비교 환경을 구성하고, 실행 정확도 등 기존 지표와 본 연구가 새로 제안하는 의도 일치 점수를 함께 측정한다. 이어 제안 프레임워크를 실제 기업 인사 데이터베이스의 테이블 구조만 가져오고 내용은 합성 데이터로 채운 환경에 적용해, 벤치마크를 넘어 실무형 데이터 구조에서도 자기교정이 효과적으로 작동하는지 검증한다.
본 연구는 의미 차원의 검증과 오류 유형별 분류라는 방법론적 기여와 더불어, 공개 벤치마크와 실제 기업 데이터베이스 환경을 아우르는 이중 검증을 수행하였다. 이를 통해 Text-to-SQL 기술의 기업 정보시스템 도입 가능성을 실증적으로 확인하였으며, SQL 전문성이 없는 현업 사용자도 자연어 기반으로 데이터에 접근할 수 있는 셀프서비스 분석 환경 구축에 실무적 시사점을 제공한다.

**주제어:** Text-to-SQL, 대규모 언어모델, 자기교정, 자연어 추론(NLI), 스키마-온리 평가
