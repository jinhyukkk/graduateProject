실험 계획서
논문: 오류 유형 인식 기반 자기교정 에이전트를 활용한 Text-to-SQL 정확도 향상 연구

1. 실험 목표
BIRD dev set 기준으로 5조건 Ablation 실험을 수행하여, 오류 유형 분류 기반 적응적 자기교정 파이프라인의 효과를 EX와 ECR로 정량화한다.

2. 실험 환경
항목
내용
모델
GPT-4o (OpenAI API)
프레임워크
LangGraph
데이터
BIRD dev set, 600개 stratified sampling
SQL 방언
SQLite
평가 지표
EX (Execution Accuracy), ECR (Error Correction Rate)

3. 5조건 Ablation 설계
조건
이름
설명
C1
No Correction
교정 없음. Schema RAG + NL2SQL만 적용. 베이스라인.
C2
Naive Retry
오류 발생 시 동일 프롬프트로 단순 재시도.
C3
Fixed CoT
유형 분류 없이 모든 오류에 CoT 교정 일괄 적용.
C4
Fixed RAG
유형 분류 없이 모든 오류에 RAG 교정 일괄 적용.
C5
Adaptive (제안)
Error Classifier로 유형 분류 후 Schema/Join/Aggregation → CoT, Condition/Logic → RAG 분기 적용.
C3, C4는 "분류 없는 고정 순서 교정" 역할로, 분류 자체의 기여(RQ2)를 분리하기 위한 비교 조건이다.

4. 평가 방법
EX는 전체 600개 기준 실행 결과 일치율로 계산한다. ECR은 C1에서 오류로 분류된 샘플만을 대상으로, 각 조건별 교정 성공률을 측정한다. 두 지표 모두 오류 유형(Schema / Join / Aggregation / Condition / Logic) 별로 분해 보고한다.
