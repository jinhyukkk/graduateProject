# 초록 v2 (개선안)

## 국문 초록 (재작성)

기업 데이터 활용 수요가 커지면서 SQL 비전문가의 자연어 질의 응답을 위한 Text-to-SQL 자기교정의 중요성이 높아지고 있다. 그러나 기존 자기교정 연구는 의미 검증을 SQL 절 단위 추론, 실행 결과 대조, SMT 동치성, NL 단위 테스트 등 SQL 쪽이나 DB 실행 결과 쪽에서만 수행해 왔고, 생성된 SQL의 의도를 자연어 공간에서 직접 대조하는 접근은 부재하다. 본 연구는 생성 SQL을 자연어로 역번역한 뒤 원 질의와 NLI 모델로 entailment·contradiction을 판정하는 자연어 공간 의도 일치 검증(Intent Consistency Checker)을 일차 차별점으로 하는 자기교정 프레임워크 SC-TSQL을 제안한다. 검증 신호는 오류 유형 진단과 결합되어, 감지된 유형별로 분기되는 전용 교정 프롬프트 템플릿을 구동하며, 이 유형 분기는 기존 연구의 공용 fixer에 유형 정보를 전달하던 방식을 확장한다. 실험은 BIRD dev에서 DIN-SQL·DAIL-SQL·CHESS를 동일 GPT-4o 백본으로 재실행하여 공정 비교하며, 추가로 기업 HR 데이터베이스의 실데이터 없이 테이블·컬럼·관계 DDL 구조만 이식하고 row는 합성 생성기로 채우는 스키마-온리 평가 프로토콜을 병행한다. 평가 지표는 Execution Accuracy, Valid Efficiency Score, Exact Match와 자체 제안 Intent Consistency Score이며, 어블레이션은 역번역 제거, NLI 제거, 유형 주입 제거, 유형 분기 제거, 반복 루프 제거의 다섯 축으로 설계된다. HR 결과는 분포 일반화가 아니라 스키마 구조가 유발하는 오류 패턴 범위로만 해석한다. 본 연구는 자연어 공간 의도 검증이라는 새로운 의미 검증 축과 유형 분기 교정, 프라이버시 친화적 스키마-온리 평가 프로토콜을 제시한다.

**주제어:** Text-to-SQL, 자기교정, 자연어 추론(NLI), 의도 일치 검증, 유형 분기 교정 프롬프트, 스키마-온리 평가, 기업 인사관리 시스템

## 변경 요약
- 일차 차별점을 "3계층 10유형 오류 분류"에서 "SQL→NL 역번역 + NLI 의도 일치 검증"으로 교체(gap_matrix에서 taxonomy는 `이미 존재`, semantic_check 축이 유일한 `진짜 gap`으로 판정됨).
- "유형별 프롬프트 주입"은 `부분 gap`이므로 "유형별 전용 템플릿 분기"로 톤 다운하여 기존 접근의 확장임을 명시.
- HR 실데이터 확장을 "향후 계획"에서 "본 실험의 스키마-온리 이식 프로토콜"로 전진 배치하고, 실데이터 없음을 숨기지 않고 한 문장 이상으로 명시.
- 베이스라인·지표·어블레이션 축을 구체적으로 기재(DIN-SQL/DAIL-SQL/CHESS, EX/VES/EM/ICS, 5축).
- 주제어에서 "오류 유형 분류" 대신 NLI·의도 일치·스키마-온리 평가 등 실제 차별점에 해당하는 용어로 교체.
