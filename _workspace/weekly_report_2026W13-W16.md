## W14 (03-30 ~ 04-05): 논문 하네스 착수 → 프로젝트2로 분기

- **03-31** — 진행률 재점검 및 마감일 확정.
- **04-01 오전** — 논문 작성 자동화 하네스 구성 시작. 선행연구·연구설계·실험분석·집필·품질검토 에이전트 팀과 감독자 구조 설계. **연구 목적** "비전문가가 자연어로 기업 DB를 조회할 수 있도록, 오류를 스스로 인식·교정하는 에이전트 구축"을 공식 선언.
- **04-01 오후** — 졸업프로젝트2 디렉토리로 전환. 실험환경 구축 하네스 착수 및 논문 초안 생성.
- **04-02** — 실험환경 하네스와 웹 대시보드 하네스를 분리 구성. 에이전트 도메인 격리 결정.
- **04-03** — 논문 완성 하네스 구성. **HR DB 스키마-온리** 방향 최초 확정(보안상 DB명·컬럼명 변형, 합성 데이터 사용).
- **04-04** — BIRD 데이터 접근 확인(DataGrip), 실험 결과 미출력 원인 디버깅, 교수님 설명용 실험 진행 방법 문서 작성.

## W15 (04-06 ~ 04-12): 초록 정제 + 베이스라인 재현 가능성 조사

- **04-06** — HRDB 스키마 추출 방법 및 실험 유의미성을 위한 최소 테이블 수 논의.
- **04-07**
  - 초록 과잉·부족 표현 점검, 영문 초록 제거, HR-DB 중심으로 재작성.
  - LLM Text-to-SQL 오류율 근거 수집.
  - 초록 개선 스킬을 활용해 선행연구 추출 → gap 분석 → 포지셔닝 → 재작성 전 파이프라인 실행. HR 스키마-온리 제약 통합.
- **04-08** — 초록 빈 수치(X.X%, Y.Y%)를 채울 실험환경 점검. DIN-SQL·DAIL-SQL·CHESS·MAC-SQL 재현 가능성 조사.
- **04-09** — NLI 검증기 단독 정밀도·재현율 파일럿 측정 환경 확인.
- **04-11 ~ 04-12** — 프로젝트 안내 문서 정비. 전체 프레임워크 실측 가능 여부 대규모 점검(334 메시지).

## W16 (04-13 ~ 04-19, 진행 중)

- **04-14** — 프로젝트 현재 상태 점검, 논문 작성 에이전트 구조 역할 확인, 주간 보고서 정리(본 문서).
- **04-16** — **전체 진행 상태 진단** 실시.
  - HRDB 현황: 557개 테이블 확보 완료이나 실험용 질의가 7건에 그쳐 논문 실험 적용 보류 → **BIRD-only 우선 경로** 확정.
  - Phase별 완료율 점검: 환경구축(60%) / 문헌조사(70%) / 구현(50%) / 실험실행(0%) / 논문집필(30%) — 핵심 블로커 3건 식별.
  - BLOCKER-1: 실험 설정 파일의 모델명이 실험 스펙과 불일치.
  - BLOCKER-2: 비교 대상 베이스라인(DAIL-SQL·MAC-SQL·Zero-shot) 실행 래퍼 미구현.
  - BLOCKER-3: 실험 결과 0건 — 논문 본문 수치 전부 미확정 상태.
  - 기술 스택 분석 및 DAIL-SQL 재현 가능성 검증 문서 작성.
- **04-17** — **파이프라인 품질 고도화·운영 안정화 구현** (커밋 3건).
  - SQL 생성 신뢰도 자동 평가 기능 추가: logprob 기반 신뢰 점수와 cross-encoder 2단계 재순위화로 더 정확한 SQL 후보 선정.
  - 파이프라인 각 단계별 처리 시간 추적 기능 추가 — 성능 병목 파악 가능.
  - 입출력 안전장치(Guardrails) 신규 구현: SQL 인젝션 차단·질의 길이 제한(입력 측), 결과 1,000행 제한·저신뢰 쿼리 경고(출력 측).
  - StateGraph 기반 파이프라인 오케스트레이터 신규 구현 — 설정 플래그 하나로 기존 방식과 교체 가능.
  - **베이스라인 래퍼 구현 완료**(BLOCKER-2 해소): DAIL-SQL·MAC-SQL·Zero-shot GPT-4o 세 가지 비교 대상을 동일 환경에서 실행할 수 있는 래퍼 작성.
  - 평가 스크립트에 신뢰도 평균 및 단계별 소요 시간 리포트 항목 추가.
  - GitHub Actions CI/CD 파이프라인 구축: 구문검사·Guardrails 단위테스트·API 스키마 검증·프론트엔드 타입 검사 4단계 자동화.
  - **데모 UI 대규모 개선**:
    - SQL 결과와 교정 이력을 항상 화면에 표시하도록 변경(기존: 토글 접기).
    - 파이프라인 진행 상황 요약 바 추가 — 스키마 링킹·SQL 생성·실행 검증·의미 검증·교정 라운드 각 단계 상태를 색상 칩으로 한눈에 표시.
    - 로딩 중 파이프라인 단계가 실시간으로 진행되는 애니메이션 추가.
    - 스키마 표시 방식을 전체 카드 뷰에서 컴팩트 태그 뷰로 변경(FK 관계는 버튼 클릭 시 토글).
    - 메뉴를 한국어로 변경.
    - 백엔드 환경변수 자동 로드 및 개발 서버 CORS 설정 보완.
  - **에이전트·스킬 체계 전면 재편**: 초록 개선 중심의 구형 에이전트/스킬을 제거하고, 논문 집필 전 과정(PM·연구자·엔지니어·분석가·집필자·검토자)을 담당하는 6인 팀 구조로 교체.
- **04-18** — 데모 UI 추가 개선 착수 — 추론 단계·SQL을 항상 표시하는 기능 보완 예정.
