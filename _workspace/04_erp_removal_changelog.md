# EnterpriseDB(ERP) 제거 변경 내역

**편집일:** 2026-04-03

## 변경 요약

EnterpriseDB(제조업 A사 ERP)를 전면 제거하고 HRM-DB(인사관리 시스템)만 기업 내부 데이터셋으로 유지.

| 섹션 | 변경 |
|------|------|
| 초록 | "복수의 실제 기업" → "실제 기업 인사관리 시스템" |
| 1.3 기여 | "복수 도메인의 실제 기업 데이터셋" → "실제 기업 인사관리 시스템 데이터셋" |
| 3.2 오류 분포 | "BIRD+EnterpriseDB 500개" → "BIRD 500개 + HRM-DB 150개"로 분리 표기. 표 1을 BIRD/HRM-DB 2열 비교표로 재구성 |
| 5.1 데이터셋 | EnterpriseDB 항목 삭제. HRM-DB 설명에서 EnterpriseDB 비교 문구 제거 |
| 5.2 표 2 | EnterpriseDB 열 삭제 (4열 → 3열 + HRM-DB) |
| 5.2 분석 | EnterpriseDB 비교 문구 모두 제거 |
| 표 3 | EnterpriseDB 교정 성공률 표 삭제. 표 3-b(HRM-DB)를 표 3으로 승격 |
| 5.3 어블레이션 | 표 4(EnterpriseDB) 삭제, 표 4-b(HRM-DB)를 표 4로 승격. 도메인 간 비교 분석 제거 |
| 5.4 반복횟수 | 표 5를 HRM-DB 기준으로 교체. EnterpriseDB 비교 문단 제거 |
| 5.5 응답시간 | HRM-DB 단독 분석으로 재작성 |
| 6.1 토론 | ERP 언급 제거. HR 도메인 단독 분석으로 간결화 |
| 6.2 한계 4 | "도메인 간 편차" → "단일 도메인에서의 검증"으로 교체 |
| 7 결론 | ERP 관련 문구 제거. 수치 HRM-DB 기준 |
| 부록 A | EnterpriseDB 비교표(표 A-2) 삭제 |

## 현재 논문의 데이터셋 구성

1. Spider (벤치마크)
2. BIRD (벤치마크)
3. HRM-DB (실제 기업 인사관리 시스템, 42테이블, 347컬럼, 150질의)
