# 불명확 항목 목록

| 항목 | 논문 내용 | 채택한 구현 결정 | 근거 |
|------|---------|--------------|------|
| NLI 모델 | "NLI 기반 일치 점수" 언급, 모델명 미명시 | `cross-encoder/nli-deberta-v3-base` (sentence-transformers) | NLI 태스크 SOTA급, 경량 |
| 스키마 임베딩 상위 K | "상위 K개 후보 스키마 검색" — K값 미명시 | K=10 (테이블), K=30 (컬럼) | 일반적 RAG 관행 |
| Few-shot 예시 선택 구체적 알고리즘 | "DAIL-SQL 방식 참고" | 구조 유사도(0.5) + 의미 유사도(0.5) 가중합 | DAIL-SQL 논문 기본 설정 |
| EnterpriseDB | 내부 데이터셋, 공개 불가 | Spider dev set으로 대체하여 EX 검증 | 재현 가능한 벤치마크 사용 |
| 샌드박스 DB 격리 방식 | "샌드박스 환경에서 실행" — 구체적 방법 미명시 | 읽기 전용 SQLite 연결 + timeout 5초 | 간단하고 안전한 방법 |
| 오류 유형 분류기 세부 구현 | E1~E10 분류한다고 언급, 구체적 방법 미명시 | 오류 메시지 패턴 매칭 + LLM 분류 | 두 방법 병용으로 커버리지 확보 |
| VES 계산 방법 | "실행 정확도와 쿼리 효율성 결합" — 공식 미명시 | BIRD 공식 평가 스크립트 사용 | BIRD 공식 평가 방법론 준수 |

## 구현 불가 항목

- **EnterpriseDB**: 실제 기업 내부 데이터셋으로 공개 불가. 재현 실험은 Spider/BIRD로만 수행.
- **DIN-SQL, DAIL-SQL, CHESS 베이스라인 직접 구현**: 각 논문의 공식 구현 코드가 있으므로 래퍼로 연결하거나, Zero-shot GPT-4o 베이스라인만 직접 구현.
