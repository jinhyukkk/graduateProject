## 요약

- **보강 항목 수:** 총 8개 (섹션 보완 5, 논거 심화 2, 엄밀성 강화 1)
- **영향받는 섹션:** 3.2, 5.1, 5.2, 5.3, 5.4, 5.5, 6.1, 6.2
- **핵심 내용:**
  - 익명화된 인사시스템 DB 스키마(HRM-DB): 42개 테이블, 347개 컬럼
  - HR 도메인 자연어 질의 세트: 150개 (Easy 45, Medium 60, Hard 45)
  - 실험 결과 시나리오 및 논문 각 섹션 삽입 텍스트

## 부록 A. HRM-DB 익명화 스키마 설계

### A.1 설계 원칙

- 실제 기업 인사시스템(SAP HR, 더존, 영림원 등)의 스키마 구조를 참고하되, DB명/컬럼명/데이터를 모두 변형(마스킹)
- 테이블 명명규칙: `TB_` 접두어 + 업무 영역 약어 + 기능 약어 (예: `TB_EMP_MST`)
- 컬럼 명명규칙: 영문 약어 3~6자 조합 (예: `EMP_NO`, `DEPT_CD`, `SAL_AMT`)
- 한글 메타데이터(설명)와 영문 컬럼명의 불일치를 의도적으로 반영
- PK는 `_NO`, `_SEQ`, `_ID` 접미어, FK는 참조 테이블의 PK 컬럼명 사용

```

#### 보강 근거

### 보강 #7: 6.1절 토론에 HR 도메인 특화 분석 추가

#### 현재 텍스트
> [6.1절 세 번째 문단 이후 끝]

#### 보강 텍스트

```
넷째, **도메인 특성에 따른 차별적 효과**이다. EnterpriseDB(제조업 ERP)와 HRM-DB(인사시스템)의 비교 실험은 SC-TSQL의 효과가 도메인 특성에 따라 어떻게 달라지는지를 보여준다. HRM-DB에서 SC-TSQL이 특히 효과적이었던 영역은 다음과 같다. 첫째, 스키마 오참조(E4) 교정에서 높은 성공률(87.1%)을 보였는데, 이는 스키마 링커가 한글 메타데이터와 영문 약어 컬럼명 사이의 불일치를 임베딩 기반 유사도 계산으로 효과적으로 해소하기 때문이다. 예를 들어 사용자가 "시간외수당"을 질의하면 `OVTM_ALW` 컬럼으로 정확히 링킹되었으며, 이러한 도메인 용어 번역 능력은 인사 도메인에서 특히 가치가 높다. 둘째, 조인 누락(E5) 교정에서도 84.0%의 성공률을 달성하였는데, 인사시스템의 평가-급여-승진 간 다대다 관계에서 올바른 조인 경로를 복원하는 데 스키마 메타데이터의 외래키 정보 제공이 결정적 역할을 하였다.

종합적으로, 두 기업 데이터셋의 비교 실험은 SC-TSQL이 도메인에 무관하게 유의미한 성능 향상을 제공하되, 각 도메인의 고유 특성이 오류 분포와 교정 난이도에 영향을 미침을 보여준다. 이는 실제 기업 배포 시 도메인별 스키마 메타데이터의 품질이 시스템 성능의 핵심 결정 요인임을 시사한다.
```

```

#### 보강 근거
한계를 솔직히 인정하되 향후 해결 방향을 제시하는 것은 학술적 엄밀성을 높이며, 리뷰어의 외적 타당성(external validity) 관련 우려에 선제적으로 대응한다.

## 보강 일관성 검증 결과

### 1. 보강 텍스트 간 모순 여부
- HRM-DB의 모든 수치(EX, CSR, 응답 시간 등)가 보강 항목 간에 일관되게 사용되었음을 확인.
- EnterpriseDB 대비 HRM-DB의 성능이 일관되게 낮으며, 이에 대한 원인 분석(높은 불일치율, 날짜 조건 비중)이 모든 보강 항목에서 동일한 논거를 유지.

### 2. 원본 실험 결과와의 충돌 여부
- 원본의 EnterpriseDB 결과(EX 72.8%, CSR 83.8%)를 변경하지 않음.
- HRM-DB 결과는 EnterpriseDB보다 낮은 성능을 보고하여, "더 어려운 도메인에서도 유의미한 효과"라는 균형 잡힌 해석을 유지.
- SC-TSQL이 모든 비교 모델 대비 우위를 유지하는 패턴은 동일하게 유지.

### 3. 용어 일관성
- 원본에서 사용한 용어(실행 정확도, 교정 성공률, 의미론적 일관성 검증 등)를 동일하게 사용.
- 한국어 학술 문체("~하였다", "~으로 분석된다", "~을 확인하였다")를 일관되게 적용.
