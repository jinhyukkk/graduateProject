# Paper Lit Search

갭 리포트를 기반으로 선행연구를 탐색하고 참고문헌을 보강한다.

## 워크플로우

### Step 1: 조사 계획 수립
1. 갭 리포트에서 선행연구 관련 갭을 추출한다
2. 각 갭에 대해 탐색 키워드를 3~5세트 준비한다
3. 논문의 기존 참고문헌 목록을 읽어 이미 인용된 연구를 파악한다

### Step 2: 체계적 탐색

**탐색 전략:**

1. **직접 키워드 탐색:** 갭과 직결되는 키워드로 WebSearch
   - 예: "self-correcting text-to-SQL 2024 2025"
   - 예: "LLM SQL error correction benchmark"

2. **역방향 탐색:** 논문이 이미 인용한 핵심 논문의 피인용(cited by)을 추적
   - 예: "papers citing DIN-SQL 2024"

3. **경쟁 연구 탐색:** 동일 벤치마크(Spider, BIRD)의 최신 리더보드 결과
   - 예: "Spider benchmark leaderboard 2025"
   - 예: "BIRD text-to-SQL state of the art"

4. **방법론 기반 탐색:** 논문이 사용하는 기법의 최신 발전
   - 예: "back-translation verification NLP 2024"
   - 예: "iterative refinement LLM code generation"

### Step 3: 논문 평가 및 선별

발견한 각 논문에 대해:
- **관련성:** 본 논문의 어떤 부분과 관련되는가 (1~5점)
- **최신성:** 출판 연도, 벤치마크 결과의 현재성
- **영향력:** 학회 랭크, 인용 수 (가능한 경우)
- **인용 필요성:** 이 논문을 인용하지 않으면 리뷰어가 지적할 가능성

관련성 3점 이상 + 인용 필요성이 높은 논문만 최종 목록에 포함한다.

### Step 4: 인용 맥락 작성

각 추천 논문에 대해:
1. 본 논문의 어디에서 인용할 수 있는지 (섹션, 문맥)
2. 어떤 맥락으로 인용하는지 (지지, 비교, 한계 지적, 방법론 차용)
3. 인용을 포함한 보강 문단 초안

### Step 5: 결과 저장
`_workspace/02_lit-researcher_references.md`에 저장한다.

## 출처 신뢰도 규칙

| 신뢰 수준 | 출처 유형 | 태그 |
|----------|----------|------|
| 높음 | ACL/EMNLP/NeurIPS/ICML 등 top-tier 학회 | (확인됨) |
| 중간 | arXiv 프리프린트, 워크숍 논문 | (프리프린트) |
| 낮음 | 블로그, 비공식 벤치마크 | (비공식) |
| 미확인 | WebSearch에서 제목만 확인, 전문 미확인 | [미확인] |

[미확인] 논문은 반드시 사용자 검증이 필요하다는 점을 명시한다.

## 주의사항

- **할루시네이션 방지:** 존재하지 않는 논문을 만들어내지 않는다. WebSearch로 확인할 수 없는 논문은 포함하지 않거나 [미확인] 태그를 붙인다.
- **인용 형식 통일:** 기존 논문의 참고문헌 형식(APA, IEEE 등)을 분석하고 동일 형식으로 새 인용을 작성한다.
- **과도한 인용 방지:** 한 갭당 최대 3~5개 논문. 양보다 질.
