# Research Positioning

진짜 gap을 논문 포지셔닝으로 번역하는 스킬. "어떤 각도로 이야기해야 가장 설득력 있고, 내 실험 환경에서 검증 가능한가" 를 스스로 결정한다.

## 언제 사용하는가

- `contribution-diff` 로 gap을 식별한 직후
- 초록의 차별점을 바꿔야 할 때
- 실험 제약(예: HR 스키마-온리) 안에서 주장 가능한 범위를 정해야 할 때

## 전제

- `_workspace/gap_matrix.md` 가 존재
- (선택) `_workspace/prior_work_table.md`, `초록_KITS.md`

## HR 스키마-온리 실험 제약

이 스킬의 **필수 제약**. 포지셔닝을 만들 때 이 제약을 외면하지 말 것.

- 실데이터 사용 불가, 스키마(테이블/컬럼/관계) 구조만 이식
- 합성 데이터 또는 공개 벤치마크로 row-level 채움
- **이 제약을 약점이 아닌 특징(privacy-preserving evaluation)으로 framing** 하는 게 포지셔닝의 한 축이 될 수 있다

## 작업 절차

### 1. Gap 읽기
`gap_matrix.md` 의 "3. 진짜 Gap 목록" 섹션만 읽으면 충분. 이것이 포지셔닝의 재료다.

### 2. 옵션 생성 (2~3개)

각 gap 또는 gap 조합에 대해 하나의 포지셔닝 옵션을 만든다. 옵션이 4개 이상이면 너무 분산된 것이므로, 유사한 것끼리 병합.

각 옵션은 다음 6필드로:

```markdown
### 옵션 {N}: {title}

- **thesis**: {2~3문장으로 핵심 주장}
- **primary_gap**: {gap_matrix에서 직접 인용}
- **novelty_strength**: {높음/중간/낮음} — {이유 한 문장}
- **feasibility_under_schema_only**:
  - 검증 가능 여부: {가능/어려움/불가능}
  - 실험 스케치: {데이터 구성, 베이스라인, 지표, 어블레이션 축}
- **review_risk**:
  - 예상 반론: {리뷰어가 물을 법한 질문 1~2개}
  - 방어 전략: {어떻게 답할지}
```

### 3. 점수화

3개 기준, 각 0~1 점:

| 기준 | 가중치 | 정의 |
|---|---|---|
| feasibility | 0.40 | HR 스키마-온리 환경에서 논문 수준의 실험이 실제로 돌아가는가 |
| novelty | 0.35 | 진짜 gap에 얼마나 직접·정확히 대응하는가 |
| defensibility | 0.25 | 예상 반론에 답할 수 있는가 |

총점 = 0.4*feasibility + 0.35*novelty + 0.25*defensibility

### 4. 자동 선택

- 총점 최고 옵션 선택
- 동률이면 feasibility가 높은 쪽
- feasibility 0 인 옵션은 아무리 novelty가 높아도 자동 탈락

### 5. 실험 설계 스케치 확장

선택된 옵션의 실험 스케치를 한 단계 더 구체화:
- **데이터**: HR 스키마를 어떻게 복제할지 (ERD 덤프, DDL 추출 등). row-level은 무엇으로 채울지 (합성? BIRD의 특정 서브셋 매핑?)
- **베이스라인**: 어떤 baseline 과 비교할지 (선행연구표에서 선택)
- **평가 지표**: Execution Accuracy, Exact Match, 의미 정합성 지표 등
- **어블레이션**: 제안 방법의 어떤 구성요소를 제거해 볼지

## 출력 포맷

파일: `_workspace/positioning_decision.md`

```markdown
# Positioning Decision

## 1. 고려된 옵션
(옵션 1 6필드)
(옵션 2 6필드)
(옵션 3 6필드, 선택사항)

## 2. 점수표
| 옵션 | feasibility | novelty | defensibility | 총점 |
|---|---|---|---|---|
| 옵션 1 | ... | ... | ... | ... |
| 옵션 2 | ... | ... | ... | ... |

## 3. 선택 결과
- **선택**: 옵션 N - {title}
- **선택 근거**: 3~5문장
- **탈락 옵션 이유**: 각 탈락 옵션 1~2문장

## 4. 선택된 포지셔닝의 실험 설계 스케치
- 데이터: ...
- 베이스라인: ...
- 평가 지표: ...
- 어블레이션: ...
- HR 스키마-온리 제약 통합 방식: ...
```

## 원칙

1. **자동 선택**: 사용자에게 되묻지 않는다. 점수표와 근거로 스스로 판단.
2. **실행가능성 우선**: 아름다운 이론보다 돌아가는 실험.
3. **HR 제약 숨기지 말 것**: 포지셔닝에 명시적으로 녹여라. 숨기면 리뷰어가 찾아낸다.
4. **탈락 옵션의 장점도 기술**: 선택 옵션만 칭찬하는 글은 신뢰가 떨어진다.

## 에러 핸들링

- `gap_matrix.md` 에 `UPSTREAM_MISSING` 또는 `NO_REAL_GAP` 플래그가 있으면:
  - `NO_REAL_GAP` 의 경우: 포지셔닝 옵션을 만들지 말고, `_workspace/positioning_decision.md` 에 "선행연구가 이미 4개 축을 모두 커버 — 초록의 차별점 재발굴 필요" 라는 결론과 함께 `NEEDS_NEW_ANGLE` 플래그 남김.
  - `UPSTREAM_MISSING` 의 경우: 동일 플래그만 남기고 종료.
- 모든 옵션의 feasibility가 0이면: `NEEDS_EXPERIMENT_REDESIGN` 플래그 + 이유 기술.
