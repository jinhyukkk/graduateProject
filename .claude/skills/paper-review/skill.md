# Paper Review — 3관점 교차 검토 + 통합 편집

검토자/편집자 역할의 실행 가이드. 한 명의 에이전트가 순차적으로 3관점을 수행한 뒤 통합한다.

## 핵심 원칙

**관점별로 기록을 분리한다.** 통합 문서에 섞지 말고 3개 파일로 먼저 쓴 뒤 마지막에 합친다. 이유: 한 관점이 다른 관점을 가리면 편집 단계에서 판정이 왜곡됨.

**상충 지적은 삭제 금지.** 방법론 검토자가 "지표 추가 필요", 논리 검토자가 "분량 초과로 지표 축소"라고 하면 **둘 다 기록**하고 편집 단계에서 연구 목적(RQ 방어) 기준으로 판정. 판정 근거를 수정 지시서에 남김.

**KITS 양식 위반 = Critical.** 분량·양식 규정은 제출 거부 사유. 타 지적보다 우선.

## 검토 순서

### 1. 방법론 검토 → `_workspace/05_review_methodology.md`

점검 항목:
- [ ] NLI 역번역 알고리즘이 수식·의사코드로 명시됐는가
- [ ] Semantic Verifier의 임계값 θ=0.75 선택 근거 명시
- [ ] 어블레이션 A1~A4가 RQ1·RQ2를 **분리 검증**하는가
- [ ] 지표(EX·VES·EM·ICS)가 중복·누락 없이 적절한가
- [ ] ICS가 EX를 대체하지 않는다는 명시가 방법·토의에 있는가
- [ ] 베이스라인이 동일 GPT-4o로 재실행됐는가 (인용 수치 사용 금지)
- [ ] 통계 검정(paired bootstrap n=1000)이 적용됐는가
- [ ] 재현성 정보: seed, temperature, config.yaml 링크
- [ ] 리뷰어 반론 3종(순환성·NLI 도메인·HR 일반화)이 방어됐는가

출력 형식:
```
### [Severity] 항목명
- 위치: 3.5절, 단락 2
- 문제: ...
- 제안: ...
- 담당: paper-writer-kr / paper-researcher / paper-analyst
```

### 2. 논리 검토 → `_workspace/05_review_logic.md`

점검 항목:
- [ ] 초록 → 서론 → RQ → 방법 → 실험 → 토의 → 결론이 **일관**된가
- [ ] 주장 = 증거 구조: 모든 성능 주장에 표·수치 참조가 있는가
- [ ] 선행연구 커버리지 (4축 모두 언급)
- [ ] RQ1·RQ2가 실험 결과로 **각각 답해졌는가**
- [ ] 기여 3~4개가 서론·결론에서 동일한 문구로 재진술되는가
- [ ] 한계가 정직하게 기술됐는가 (HRDB 스키마-온리, 단일 LLM)
- [ ] 과잉 주장 검색: "최초"/"SOTA"/"완벽"/"모든"/"혁신"

### 3. 형식·인용 검토 → `_workspace/05_review_format.md`

점검 항목:
- [ ] KITS 분량 규정 준수 ([05_KITS_submission_guidelines.md](../../../_workspace/05_KITS_submission_guidelines.md) 확인)
- [ ] 섹션 넘버링 (1, 1.1, 1.1.1 스타일)
- [ ] 표·그림 캡션 완전성 (번호·제목·지표·데이터셋)
- [ ] 참고문헌 스타일 일관 (KITS 양식)
- [ ] 프리프린트 → 최종 게재 정보 치환 (arXiv만 인용된 논문 확인)
- [ ] BibTeX 파일 `_workspace/references.bib` 존재 및 중복 없음
- [ ] 한국어 학술 톤 (영어 직역·구어체 제거)
- [ ] 오탈자·띄어쓰기

### 4. 통합 편집 → `_workspace/06_revision_guide.md` → `_workspace/07_paper_final.md`

단계:
1. 3개 검토 파일을 읽고 지적을 **Critical / Major / Minor**로 분류
2. 상충 지적을 RQ 방어 기준으로 판정, 판정 근거 명시
3. 수정 지시서 작성 (`06_revision_guide.md`): 각 지적에 대해 "수용/기각/조건부 수용" 결정
4. 초안에 수정을 적용해 `07_paper_final.md` 생성
5. 분량 초과 시 압축 순서: Introduction → Discussion → Future Work. Method·Experiment는 압축 후순위.

## 참고문헌 관리

별도 파일 `_workspace/references.bib` 유지:
- 형식: BibTeX
- 중복 감지: key 중복 시 병합 (연도·저자 일치 확인)
- 프리프린트: `@misc{..., note={arXiv:...}}` → 최종 게재 확인 시 `@inproceedings` 또는 `@article`로 변경
- KITS 양식 렌더링: 본문 인용 스타일에 맞게 변환 (각주/괄호/번호)
