# Paper Argument Building

갭 리포트를 기반으로 논문의 논거를 보강하고 약한 섹션을 강화한다.

## 워크플로우

### Step 1: 원저자 톤 분석
논문 전문을 읽고 다음을 파악한다:
- **문체:** 능동태/수동태 비율, 문장 길이, 전문 용어 수준
- **서술 패턴:** 주장 제시 방식, 근거 인용 방식, 전환 표현
- **어조:** 자신감 수준 ("~임을 보인다" vs "~일 가능성이 있다")
- **언어:** 한국어/영어, 혼용 비율

이 분석을 기반으로 보강 텍스트의 톤을 맞춘다.

### Step 2: 갭별 보강 전략 수립
갭 리포트의 각 항목에 대해 보강 유형을 결정한다:

| 갭 유형 | 보강 전략 |
|--------|----------|
| 근거 부족 | 실험 결과 재해석, 선행연구 인용 추가, 논리적 추론 보강 |
| 설명 부족 | 개념 정의 추가, 예시/비유 제공, 단계별 설명 확장 |
| 논리 단절 | 전환 문장 삽입, 인과관계 명시, 전후 참조 추가 |
| 반론 미대응 | 예상 반론 제시 + 선제적 반박 |
| 일반화 과잉 | 주장 범위 한정, 조건부 서술로 전환 |
| 비교 부족 | 기존 방법과의 체계적 비교 추가 |

### Step 3: 보강 텍스트 작성

각 보강 항목에 대해:

1. **현재 텍스트 인용:** 기존 논문에서 해당 부분을 정확히 인용
2. **보강 텍스트 작성:** 주장→근거→결론 삼단 구조로 작성
3. **삽입 지시:** 정확한 위치 (섹션.문단, "~한다." 뒤에 삽입)
4. **보강 근거:** 왜 이 보강이 필요한지 1~2문장으로 설명

### Step 4: 일관성 검증
모든 보강 텍스트를 취합한 후:
- 보강 텍스트끼리 모순이 없는지 확인
- 원본 논문의 실험 결과와 충돌하는 주장이 없는지 확인
- 용어 사용이 원본과 일관된지 확인

### Step 5: 결과 저장
`_workspace/02_argument-builder_enhancements.md`에 저장한다.

## 보강 품질 기준

**좋은 보강:**
- 기존 주장을 더 설득력 있게 만든다
- 리뷰어의 예상 질문에 선제적으로 답한다
- 원문과 자연스럽게 어우러진다
- 실험 결과가 뒷받침하는 범위 내에서만 주장한다

**나쁜 보강:**
- 기존 내용을 단순 반복/패러프레이즈
- 과장되거나 근거 없는 주장 추가
- 원문과 문체가 이질적
- 논문의 범위를 넘어서는 주장

## 섹션별 보강 포인트

### 서론 보강
- 연구 동기의 시급성/중요성 강화
- 기존 접근법의 한계를 더 명확하게
- 연구 기여의 차별점을 더 선명하게

### 관련 연구 보강
- 비판적 분석 추가 (단순 나열 → "A는 ~했으나, ~라는 한계가 있다. 반면 본 연구는...")
- 연구 흐름의 논리적 서사 구축

### 방법론 보강
- 설계 결정의 근거 추가 ("왜 K=3으로 설정했는가")
- 알고리즘 의사코드, 수식 보완
- 기존 방법과의 차이점 명시

### 실험/토론 보강
- 결과 해석의 깊이 강화 ("왜 이런 결과가 나왔는가")
- 예상 반론에 대한 선제적 대응
- 위협 요인(threats to validity) 분석
- 실용적 시사점(practical implications)

### 결론 보강
- 기여의 학술적·실용적 의의 강조
- 향후 연구 방향의 구체화
