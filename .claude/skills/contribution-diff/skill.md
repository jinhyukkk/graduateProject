# Contribution Diff

**초록이 주장하는 차별점이 실제로 선행연구의 gap인지** 를 정량·정성적으로 판정하는 스킬. "주장된 novelty ≠ 실제 novelty" 라는 원칙에서 출발한다.

## 언제 사용하는가

- 선행연구 추출표가 이미 있고, 그에 대비해 자기 논문의 주장을 검증할 때
- 초록/Introduction이 내세운 기여점의 진정성을 사전 검사할 때
- 리뷰어의 "이거 이미 있잖아요?" 반박을 미리 차단하고 싶을 때

## 전제

- `_workspace/prior_work_table.md` 가 존재 (없으면 `extract-prior-work` 스킬을 먼저 실행)
- 비교 대상 초록 파일이 존재 (기본: `초록_KITS.md`)

## 4개 비교 축 (Text-to-SQL 자기교정 전용)

이 스킬은 다음 4개 축으로만 대조한다. 축 자체는 `초록_KITS.md` 가 내세우는 핵심 기여에서 역으로 도출된 것이므로, 다른 초록에 재사용할 경우 축을 재정의해야 한다.

| 축 | 정의 | 판정 기준 |
|---|---|---|
| `taxonomy` | 오류 분류 체계 | 3계층 10유형 수준까지: 1.0 / 부분 유형 분류: 0.5 / 없음: 0 |
| `semantic_check` | 의미 검증 (역번역+NLI) | SQL→NL 역번역 + NLI 의도 비교: 1.0 / 실행 외 추가 검증만: 0.5 / 실행 검증뿐: 0 |
| `type_injection` | 유형별 프롬프트 주입 | 감지된 오류 유형이 프롬프트 변수로 주입: 1.0 / 일반 피드백만: 0.5 / 주입 없음: 0 |
| `iterative_loop` | 반복 교정 루프 | 유형 진단 갱신 + 재교정 루프: 1.0 / 단순 재시도: 0.5 / 단발: 0 |

## 작업 절차

### 1. 입력 로드
- `_workspace/prior_work_table.md` 파싱 → 논문 리스트
- `초록_KITS.md` 읽기 → 주장 추출

### 2. 매트릭스 채우기
각 논문 × 4축 셀에 점수 부여. 근거는 `prior_work_table.md` 의 어느 필드에서 왔는지 명시. 추측 금지 — 표에 없는 내용으로 점수를 매기지 말 것.

### 3. 초록 주장 판정
초록에서 다음 패턴의 문장을 추출:
- "~를 제안한다"
- "새로운 ~"
- "기존 접근은 ~ 못한다"
- "본 연구는 ~" 로 시작하는 기여 문장

각 주장에 대해:
- 어느 축에 해당하는지 매핑
- 해당 축의 매트릭스 컬럼을 집계 (예: "9편 중 3편이 0.5 이상")
- 판정: `진짜 gap` / `부분 gap` / `이미 존재`

판정 규칙:
- 9편 모두 0 → `진짜 gap`
- 9편 중 일부가 0.5, 아무도 1.0 없음 → `부분 gap` (기존 접근의 확장으로 포지셔닝 가능)
- 누군가 1.0 → `이미 존재` (novelty 주장 불가)

### 4. 진짜 Gap 목록화
`진짜 gap` + `부분 gap` 만 추려 3개 이하로 정리. 각 gap에 **"왜 의미 있는가(implication)"** 한 문장 필수.

## 출력 포맷

파일: `_workspace/gap_matrix.md`

```markdown
# Gap Matrix: {초록 제목} vs {선행연구 폴더}

## 1. 커버리지 매트릭스

| 논문 | taxonomy | semantic_check | type_injection | iterative_loop |
|---|---|---|---|---|
| DIN-SQL | 0 (언급 없음) | 0 (실행만) | 0 | 0.5 (재시도만) |
| CHESS | ... | ... | ... | ... |
| ...

## 2. 초록 주장 vs 실제 gap

### 주장 1: {원문 인용}
- **매핑 축**: {축}
- **선행연구 커버리지**: {요약}
- **판정**: {진짜 gap / 부분 gap / 이미 존재}
- **이유**: {1~2문장}

### 주장 2: ...

## 3. 진짜 Gap 목록

1. **{gap 이름}** — {설명}. *Implication*: {왜 의미 있는가}
2. ...

## 4. 초록에 포함된 추가 주장 (축 밖)
(해당 없음 또는 기타 주장 기술)
```

## 원칙

1. **가차 없이 판정** — "부분적으로 비슷" 은 `부분 gap` 이지 `진짜 gap` 이 아니다. 스스로에게 관대하면 리뷰어가 대신 가혹해진다.
2. **근거는 반드시 prior_work_table.md 에서** — 원 PDF를 다시 읽지 말 것. 표에 없으면 "없음" 으로 처리.
3. **gap 인플레이션 금지** — 3개 초과로 gap을 늘리지 말 것. 많으면 논문의 초점이 흐려진다.

## 에러 핸들링

- `prior_work_table.md` 에 `NO_INPUT` 이 있으면: `gap_matrix.md` 에 `UPSTREAM_MISSING` 만 남기고 종료.
- 초록의 주장이 4개 축에 매핑되지 않는 경우: 섹션 4에 기록하되 축을 확장하지 말 것.
- 판정이 모두 `이미 존재` 로 나오면: Gap 목록은 비우되, 섹션 3에 `NO_REAL_GAP` 플래그 남김. 이것은 정직한 결론이며, 후속 positioning-advisor가 재검토하게 된다.
