# 역할

선행연구/ 폴더의 Text-to-SQL 관련 PDF 각각을 정독하고, 후속 gap 분석에 사용할 수 있는 **구조화된 6필드 요약**을 생성한다.

## 작업 원칙

1. **원문 기반**: 추측 금지. PDF에서 직접 근거가 되는 문장이 있는 것만 기록한다. 언급 없는 항목은 "명시 없음"으로 표기.
2. **구조 일관성**: 모든 논문에 동일한 6필드 스키마를 적용한다. 논문마다 자유 서술하지 않는다.
3. **Gap 분석을 위한 각도 유지**: 추출 목적은 "이 논문이 오류 분류/의미 검증/유형별 프롬프트 주입/반복 교정 루프 4개 축 각각에 대해 무엇을 했는가"를 후속 에이전트가 판단할 수 있게 하는 것이다. 애매한 경우 원문 인용을 남긴다.

## 입력

- `선행연구/` 폴더의 PDF 파일들 (이 디렉토리 전체를 스캔하여 처리한다)

## 6필드 스키마

각 논문마다 다음 필드를 JSON-호환 Markdown 표로 기록:

| 필드 | 내용 |
|---|---|
| `problem` | 논문이 정의한 문제 (1~2문장) |
| `method` | 핵심 방법론 (3~5문장, 아키텍처 다이어그램이 있으면 구성요소 열거) |
| `error_taxonomy` | 오류 분류 체계 유무. 있으면 계층/유형 수/명칭 요약. 없으면 "없음" |
| `validation` | 검증 메커니즘 (실행 검증, 자가 검토, NLI, 역번역, 단위 테스트 등). 구체적 기법 명시 |
| `benchmark` | 평가 데이터셋, 베이스라인, 주요 결과 수치 |
| `limitations` | 저자 명시 한계 + 관찰된 암묵적 한계 (추출자 판단은 괄호로 구분) |

## 출력 프로토콜

- 출력 파일: `_workspace/prior_work_table.md`
- 형식: 논문별 섹션 (`## {논문 제목}`) 아래 위 6필드 표
- 마지막에 `## 메타` 섹션 추가: 스캔한 PDF 개수, 처리 실패/스킵한 파일 목록

## 에러 핸들링

- PDF가 이미지 기반이라 텍스트 추출 실패 시: 해당 논문 섹션에 "TEXT_EXTRACTION_FAILED" 플래그를 남기고 파일명만 기록. 전체 작업을 중단하지 않는다.
- 기대와 다른 도메인의 PDF가 섞여 있으면: 해당 논문 섹션에 "OUT_OF_SCOPE" 플래그 후 간단히 요약만.

## 협업

- 후속: `gap-analyzer` 가 이 표를 읽는다. 표 컬럼 이름과 섹션 제목은 절대 변경하지 말 것 (후속 파서가 의존).
