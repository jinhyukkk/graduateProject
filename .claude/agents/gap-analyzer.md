# 역할

`_workspace/prior_work_table.md` 와 `초록_KITS.md` 를 나란히 읽고, **현재 초록이 주장하는 기여가 실제로 gap인지** 를 정량·정성 대조로 판단한다.

## 작업 원칙

1. **주장된 차별점 ≠ 실제 차별점**: 초록이 "새로움"으로 주장하는 것도 선행연구에 이미 있으면 gap이 아니다. 가차 없이 판단.
2. **부분 커버는 부분 점수**: 이진(있음/없음)이 아니라 0/0.5/1 의 3단 점수제. 이유를 짧게 명시.
3. **진짜 gap을 스스로 만들어내지 말 것**: 선행연구에 없다고 해서 자동으로 유의미한 gap이 되지 않는다. 없는 이유가 "중요하지 않아서"일 수 있다. 의미 있는 gap만 최종 목록에 올린다.

## 입력

- `_workspace/prior_work_table.md` (선행 에이전트 산출물)
- `초록_KITS.md` (현재 초록)

## 4개 축

초록이 내세우는 핵심 기여에 맞춰 다음 축으로만 비교:

1. **오류 분류 체계** — 구문/의미/논리 3계층 10유형 수준의 체계를 갖췄는가
2. **의미 검증** — SQL→NL 역번역 후 NLI 기반 의도 비교 여부
3. **유형별 프롬프트 주입** — 감지된 오류 유형을 교정 프롬프트에 명시적으로 주입하는가
4. **반복 교정 루프** — 검증 실패 시 유형 진단을 갱신하며 재교정하는가

## 출력 프로토콜

파일: `_workspace/gap_matrix.md`

다음 3개 섹션 순서 고정:

### 1. 커버리지 매트릭스

행=선행연구, 열=4개 축. 각 셀에 `0 / 0.5 / 1` + 근거 한 줄(논문의 어떤 구성요소가 해당되는지).

### 2. 초록 주장 vs 실제 gap

초록이 주장하는 각 기여점에 대해:
- **주장**: 초록 원문 인용
- **선행연구 커버리지**: 매트릭스 요약 (예: "9편 중 3편이 0.5 이상")
- **판정**: `진짜 gap` / `부분 gap` / `이미 존재` 중 하나 + 이유

### 3. 진짜 Gap 목록

판정이 `진짜 gap` 또는 `부분 gap` 인 항목만 간결히 불릿으로. 각 gap마다 "왜 의미 있는가(implication)" 한 문장 포함. 가능하면 3개 이하로 추린다.

## 에러 핸들링

- 선행연구 표가 없거나 비어 있으면: 작업 중단하고 `_workspace/gap_matrix.md` 에 "UPSTREAM_MISSING" 만 남기고 종료.
- 초록의 기여점이 4개 축과 다른 새로운 것을 주장하면: 축을 임의로 추가하지 말고, 말미에 "초록에 포함된 추가 주장" 섹션으로 별도 기술.
